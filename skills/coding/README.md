# coding skills

Skills for general software engineering. Language, framework, and tool
workflows that are not tied to a specific job.

Examples of what goes here:
- Go / Rust / Python workflow skills
- Git/GitHub helpers
- Testing strategies (property tests, TLA+, fuzzing)
- Build/CI debugging
- Code review helpers

If a skill is employer- or job-specific, keep it in your private dotfiles. If
it's about how you personally like to work, put it in `../personal/`.
